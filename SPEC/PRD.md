## 3. 项目背景

销售、采购、工程等业务人员在处理报价单后，需要根据公司合同模板手工整理合同字段、录入设备明细、生成合同文件并上传钉盘。该过程依赖人工复制和校对，容易出现字段遗漏、格式不一致、合同版本散落等问题。

合同生成助手面向钉钉内使用场景，提供从报价单上传、文本解析、字段识别、人工确认到合同生成和钉盘交付的一体化流程，降低重复录入成本，并让合同产物集中沉淀到钉盘。

## 4. 产品目标

- 支持用户上传报价单文件，并自动解析报价单中的文本、表格或图片内容。
- 根据所选合同模板识别合同字段，展示给用户确认和补充。
- 基于确认后的字段生成标准 Word 合同文档。
- 将生成的合同上传至钉盘，并在前端提供合同文件下载入口。
- 支持用户在前端创建多个处理任务，便于连续处理多份报价单。
- 支持字段识别后按乙方名称实时只读查询用友 YonBIP 供应商档案，回填乙方抬头信息。

## 5. 用户角色

| 角色 | 说明 | 核心诉求 |
| --- | --- | --- |
| 业务用户 | 销售、采购、工程等需要根据报价单生成合同的人员 | 快速上传报价单，确认字段后生成合同并下载钉盘文件 |

V1 仅面向业务用户，不包含独立后台、资源配置和审计查询页面。

## 6. 核心业务流程

1. 用户在钉钉中打开合同生成助手。
2. 系统完成钉钉免登校验，进入上传页面。
3. 用户选择合同模板并上传报价单文件。
4. 系统解析报价单文本和表格内容，返回可编辑的文本预览。
5. 用户检查解析结果，可补充额外说明信息。
6. 系统根据报价单文本、额外说明和合同模板识别合同字段。
7. 用户确认字段识别结果；未识别字段在界面中提示，用户确认后生成合同时留空，便于后续在 Word 合同中手工补充。
8. 用户点击生成合同。
9. 系统渲染 Word 合同文档并上传至钉盘。
10. 前端展示合同生成结果，并通过 FC 后端下载钉盘合同文件，提示用户保存到浏览器或钉钉客户端默认下载目录。

字段确认环节应在识别出乙方名称后实时查询用友 YonBIP 供应商档案，用用友返回的主档和默认银行信息覆盖合同中的乙方抬头字段。查询失败、未命中、命中多条或用友抬头信息不完整时不阻塞报价单任务流程，前端提示用户到用友系统补充供应商抬头信息或先手动填写。

## 7. 功能需求

### 7.1 钉钉免登

- 系统应在用户进入页面时检查当前鉴权状态。
- 在钉钉客户端内，系统应通过钉钉免登码换取用户身份并建立会话。
- 未完成鉴权时，用户不可提交报价单或生成合同。
- V1 应完成正式鉴权链路：前端通过钉钉客户端 JSAPI SDK 获取免登授权码，FC 后端使用钉钉官方新版服务端 SDK 完成服务端换取和会话协同，并基于短期访问凭证处理业务请求。

### 7.2 报价单上传

- 用户应先选择合同模板；报价单文件为可选项。
- 不上传报价单时，用户跳过解析与 AI 字段识别，直接在字段确认稿中手工补充全部合同字段后生成合同。
- 上传报价单时，按既有流程解析文本并由 AI 识别字段。
- 支持的报价单格式：
  - `.xlsx`
  - `.xls`
  - `.pdf`
  - `.jpg`
  - `.jpeg`
  - `.png`
  - `.bmp`
  - `.gif`
  - `.tif`
  - `.tiff`
  - `.webp`
- 报价单不会以 TXT 作为业务输入，V1 不将 TXT 作为正式支持格式。
- 图片报价单应通过阿里云 OCR 识别文本和表格内容，再进入后续字段识别流程。
- 上传成功后，系统应生成上传记录，并返回文件标识、原始文件名、文件大小等信息。
- 上传失败时，系统应给出明确错误提示，前端允许用户重新提交。

### 7.3 报价单解析与文本确认

- 系统应根据文件类型抽取报价单文本和表格内容。
- 对 Excel 文件，应保留工作表信息和表格结构。
- 对 PDF 文件，应尽量提取页面文本和表格内容。
- 对图片文件，应调用阿里云 OCR 识别文本和表格内容。
- 解析结果应展示为可编辑文本，用户可以修正识别错误或删除无关内容。
- 用户可填写额外说明，用于补充报价单中缺失但合同生成需要的信息。

### 7.4 合同模板选择

V1 支持以下合同模板：

| 模板标识 | 模板名称 |
| --- | --- |
| `caigouhetong` | 采购合同（通用设备） |
| `nonStandardNoInstall` | 设备采购合同（不含安装） |
| `nonStandardWithInstall` | 设备采购合同（含安装） |
| `annualFramework` | 年度采购框架合同 |
| `professionalSubcontract` | 专业工程分包合同 |
| `laborSubcontract` | 劳务分包合同（清包工） |
| `simpleContract` | 简易合同（产品购销） |

每个模板应维护独立的字段契约，用于约束字段识别结果和合同渲染。

### 7.5 字段识别与确认

- 有报价单时，系统应结合报价单解析文本、用户补充说明和所选模板字段契约，识别合同字段。
- 无报价单时，系统应直接展示所选模板的字段确认稿，由用户手工填写全部字段。
- 非附件模式下，含设备/产品明细表的模板应在字段确认稿中支持单价、数量、小计联动：单价 × 数量 = 小计；用户修改单价或数量时，小计自动计算。
- 字段识别结果应按合同字段顺序展示，方便用户逐项确认。
- 已识别字段应展示识别值。
- 未识别字段应在字段确认界面明确标记为待填写。
- 用户确认字段后，才能进入合同生成步骤；用户确认后仍缺失的字段在生成的 Word 合同中应渲染为空白，不再写入待填写占位文本。

### 7.6 合同生成

- 系统应基于所选 Word 模板和用户确认后的字段生成 `.docx` 合同文件。
- 合同文件名应包含可识别的合同信息或生成时间，避免同名覆盖。
- 生成过程中，前端应展示处理日志或进度提示。
- 生成失败时，系统应保留当前任务上下文，向用户展示失败原因，并允许用户修改信息后重试。

### 7.7 钉盘上传与下载

- 合同生成成功后，系统应通过钉盘官方新版 SDK 将合同文件上传至配置的钉盘目录。
- 上传成功后，系统应返回钉盘文件信息和前端可用的下载入口。
- 前端应通过 FC 下载接口获取合同文件，并提示用户文件会保存到浏览器或钉钉客户端默认下载目录；如系统弹窗提示，用户可选择目标保存位置。
- 钉盘上传失败时，前端应提示失败原因，并允许用户重试生成或重新提交。

### 7.8 多任务处理

- 用户可以在前端创建多个合同生成任务。
- 每个任务包含独立的报价单、合同模板、解析文本、字段识别结果和生成结果。
- 前端应展示任务列表，用户可以切换查看不同任务的状态和日志。
- V1 的多任务能力以当前浏览器页面内任务为主，不包含服务端任务持久化、跨设备恢复和批量打包下载。
- 为避免页面状态过多，前端应限制未完成任务数量；达到上限时，应提示用户先完成或删除任务。

### 7.9 用友供应商实时抬头回填

- 字段识别完成后，FC 后端应读取 `supplierName` 并调用用友 YonBIP 供应商档案自定义分页查询接口，按供应商名称精确查询主档、银行子表和联系人子表。
- 查询请求应使用 `condition.simpleVOs = [{ field: "name", op: "eq", value1: supplierName }]`，并通过 `partParam.vendorbanks.data = "*,openaccountbank.name"` 请求银行账户信息，通过 `partParam.vendorcontactss.data = "*"` 请求联系人信息。
- 系统只在命中唯一可用供应商时自动回填；未命中、多条命中、接口失败时不阻塞字段确认和合同生成。
- 用友返回的乙方抬头信息作为权威数据，应覆盖合同中的乙方名称、统一社会信用代码、地址、电话、开户行、银行账号、乙方代表姓名、乙方代表电话、乙方代表邮箱等乙方抬头字段。
- `vendorbanks` 中优先选择 `defaultbank=true` 且 `stopstatus=false` 的银行账户；否则选择第一条未停用账户。
- `vendorcontactss` 中优先选择 `defaultcontact=true` 的联系人；否则选择第一条联系人。若分页接口未返回联系人子表，后端可按唯一供应商补查详情接口。
- 若用友未返回税号、地址、电话、开户行、银行账号、联系人等合同需要的抬头字段，前端应提示用户到用友系统补充供应商抬头信息；用户仍可先手动填写并继续生成合同。
- 系统不生成、不下载、不上传供应商缓存文件，不在本地或钉盘长期保存供应商档案。

### 7.10 Excel 表格处理方式

- Excel 报价单默认按合同模板识别表格并填入合同。
- 字段识别前用户可显式勾选“将表格作为附件”：未勾选时使用 `template`，勾选时使用 `attachment`，只识别主字段并把 Excel 原表追加到合同末尾。
- 字段识别和合同生成必须使用同一个 `tableMode`，避免确认稿和最终 Word 不一致。

## 8. 任务状态

| 状态 | 说明 |
| --- | --- |
| 上传中 | 正在提交报价单文件 |
| 解析中 | 正在提取报价单文本和表格内容 |
| 待确认文本 | 报价单解析完成，等待用户确认或编辑文本 |
| 字段识别中 | 正在根据模板识别合同字段 |
| 待确认字段 | 字段识别完成，等待用户确认 |
| 合同生成中 | 正在生成合同并上传钉盘 |
| 已完成 | 合同已生成并可下载 |
| 失败 | 当前任务处理失败，可按失败步骤重试 |

## 11. 非功能需求

### 11.1 易用性

- 用户应能在同一页面完成上传、确认、生成和下载流程。
- 关键步骤应有明确状态提示，避免用户误以为页面无响应。
- 失败信息应尽量使用业务可理解语言，减少纯技术错误暴露。

### 11.2 安全性

- 正式环境应依赖钉钉免登识别用户身份。
- 钉钉免登和钉盘文件能力应统一使用官方新版 SDK，不再新增旧版接口或手写 HTTP 调用。
- 上传文件和生成合同应按最小必要原则处理。
- 合同生成完成后，系统应清理本地临时文件，减少敏感文件留存。
- 业务接口应校验服务端签发的短期访问凭证。
- 阿里云 OCR、LLM、用友 YonBIP 和钉盘访问密钥仅允许保存在服务端运行环境中。

### 11.3 兼容性

- 页面主要面向钉钉 H5 使用场景。
- 报价单解析优先保障 Excel、PDF 和常见图片格式。
- 对复杂 PDF 表格、低清晰度图片、倾斜或遮挡严重的扫描件，V1 应提示用户校对 OCR 结果。

### 11.4 可维护性

- 合同模板字段应通过字段契约维护，避免将模板字段硬编码到前端。
- 新增合同模板时，应同步提供 Word 模板和字段契约。
- 处理日志应覆盖上传、解析、字段识别、合同生成和钉盘上传等关键阶段。

## 12. 异常处理

| 场景 | 系统行为 | 用户提示 |
| --- | --- | --- |
| 未登录或会话失效 | 阻止提交并重新发起鉴权 | 请在钉钉中重新打开应用或刷新页面 |
| 文件格式不支持 | 拒绝解析 | 当前版本支持 PDF、Excel 或图片报价单 |
| 文件内容为空 | 解析失败 | 未解析到有效报价单内容，请检查文件 |
| OCR 识别失败 | 保留上传任务并返回失败原因 | 图片识别失败，请检查图片清晰度后重试 |
| 字段识别失败 | 保留已解析文本 | 请检查报价单文本或补充说明后重试 |
| 字段缺失 | 字段确认界面标记为待填写，生成合同中留空 | 请确认缺失字段，必要时在合同中手工补充 |
| 用友抬头查询失败 | 保留字段识别结果，不影响合同生成主流程 | 未能从用友读取乙方抬头，请手动补充或稍后重试 |
| 合同生成失败 | 保留任务上下文并返回失败原因 | 合同生成失败，请根据原因调整字段后重试 |
| 钉盘上传失败 | 生成流程失败或不返回下载入口 | 合同上传钉盘失败，请稍后重试 |
| 钉盘下载失败 | 保留已生成任务和钉盘文件信息 | 合同下载失败，请稍后重试或到钉盘目录手动下载 |

## 13. 测试验收点

### 13.1 功能测试

- 钉钉免登成功后，可以进入报价单上传流程。
- 上传 `.xlsx`、`.xls`、`.pdf`、`.jpg`、`.jpeg`、`.png`、`.bmp`、`.gif`、`.tif`、`.tiff`、`.webp` 文件后，可以进入文本确认步骤。
- 不支持的文件格式会返回明确错误提示。
- 图片报价单应通过阿里云 OCR 生成可编辑的解析文本。
- 用户编辑解析文本后，字段识别应基于编辑后的文本执行。
- 识别缺失字段时，前端应明确标记待填写项；用户确认生成后，合同对应位置应为空白。
- 每个合同模板均可完成合同生成流程。
- 合同生成成功后，前端可下载钉盘合同文件，并提示保存位置。
- 字段识别出乙方名称后，系统可按名称实时查询用友供应商档案并覆盖乙方抬头字段。
- 用友返回抬头信息不完整时，前端应提示用户到用友系统补充供应商档案，且不阻塞当前任务继续处理。
- 用友查询失败、未命中或命中多条时，前端应展示可理解提示，不影响已有报价单任务继续处理。
- 合同生成失败时，前端应展示失败原因并保留任务上下文。
- 失败任务可以按当前失败步骤重试。

### 13.2 兼容性测试

- 验证钉钉客户端内免登流程。
- 验证常见 Excel 报价单的多工作表解析。
- 验证普通文本 PDF 的内容提取。
- 验证清晰图片报价单的 OCR 文本和表格识别。
- 验证浏览器页面内多个未完成任务的切换和状态保持。

### 13.3 安全与清理测试

- 未鉴权用户无法上传报价单或生成合同。
- 合同生成成功后，本地临时报价单和合同文件应被清理。
- 系统不应在本地或钉盘长期保存供应商档案缓存文件。
- 业务接口应拒绝无效或过期的短期访问凭证。
- 前端请求中不应出现阿里云 OCR、LLM、用友 YonBIP 或钉盘访问密钥。

## 14. V1 暂不包含范围

- 独立后台、资源配置和任务审计查询。
- 服务端任务调度。
- 服务端批量任务组、跨设备任务恢复和批量 ZIP 下载。
- 平台级弹性伸缩、熔断、监控大盘和告警。
- 合同 PDF 输出。
- 固定可用性、任务失败率等平台级服务指标承诺。

## 15. 后续规划

- 提升低清晰度扫描件、复杂表格图片和多页图片的 OCR 识别稳定性。
- 支持服务端任务持久化，用户刷新或换设备后可继续查看任务。
- 支持钉盘目录按日期、用户或项目自动归档。
- 支持钉盘上传失败自动重试。
- 支持一次选择多个报价单并自动创建多个任务。
- 支持合同生成结果的结构化记录，便于后续查询和统计。
