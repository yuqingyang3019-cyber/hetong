# AGENTS.md

## 项目协作规则

- 以 `SPEC/` 为准。实现产品、架构、接口、鉴权、钉钉、钉盘、OCR、合同生成相关需求前，先阅读对应的 `SPEC/` 文档。
- 代码必须对齐最新 PRD、架构文档和接口文档。代码与 `SPEC/` 不一致时，默认按 `SPEC/` 调整实现，除非用户明确修改需求。
- 不要添加大范围兜底逻辑、兼容分支、重复流程或未被需求要求的备用路径。
- 需求变化后，直接删除已经用不到的代码路径、UI 状态、接口调用、测试和文档，不要把废弃逻辑留在代码里。
- 优先写简单、直接、符合当前产品范围的代码。不要提前加入 `SPEC/` 没有要求的抽象、企业级能力或平台化设计。
- 钉钉登录和钉盘文件能力必须遵循当前 `SPEC/` 方向：使用官方新版 SDK，不新增旧版 OAPI 或手写 HTTP 集成。
- 完成实质修改后，只做轻量语法或文档检查，除非用户明确要求运行环境级验证。验证通过后，提交并推送相关变更。
